Directory Structure Notes

  • The structure follows the recommended React Native organization with separate folders for screens, components, and services
  • Navigation is organized by app section (auth, main, profile) for better maintainability
  • Redux store follows the standard action/reducer/selector pattern
  • Firebase configuration is isolated in the config directory
  • Utility functions are grouped by domain (dates, validation, etc.)
  • Tests mirror the application structure for easier correlation
